OpenClaw Overtakes React as GitHub’s Most-Starred Project

OpenClaw has surpassed React to become the most-starred software project on GitHub, accumulating over 190,000 stars in just 14 days since its launch.

By Markus Kasanmascheff
March 3, 2026 11:39 am CET
TL;DR
  • Milestone: OpenClaw has surpassed React to become GitHub’s most-starred software project, a feat it accomplished in under four months of existence.
  • Record Growth: The open-source AI agent framework gained 190,000 stars in its first 14 days, making it the fastest-growing repository in GitHub history.
  • Security Warning: Security researchers have identified prompt injection vulnerabilities in OpenClaw that could allow compromised agents to take unauthorized actions on users’ accounts.

React dominated GitHub’s star rankings for years. OpenClaw erased that lead in four months, surpassing React as GitHub’s top-starred software project. This marks the first time React’s top ranking had been ended, by a framework that did not exist four months prior.

OpenClaw, an open-source AI agent framework built by a single developer, accumulated in months what established projects spent decades building.

The Record-Breaking Ascent

The numbers behind that displacement are equally striking. OpenClaw became the fastest-growing open-source repository in GitHub history, amassing 190,000 stars in its first 14 days alone. For comparison, Kubernetes has 120,000 stars after nearly a decade; the Linux kernel has 195,000 after more than 30 years of accumulation.

Its leaderboard climb came in rapid stages. OpenClaw first surpassed Linux to claim 14th place on GitHub’s all-time rankings, crossing 224,000 stars against Linux’s 218,000. At the time, React stood at 243K stars, a gap OpenClaw closed within days.

Star History author Tianzhou noted in an earlier article that “at this rate, it might not hold that lead for long.” Within days, the prediction proved accurate, as OpenClaw continued accumulating stars at a pace that outstripped any prior open-source project on the platform.

What OpenClaw Does

Behind those leaderboard movements is a project with a surprisingly brief history. OpenClaw was built by Austrian developer Peter Steinberger, though the project went through two renamings before settling on its current identity. It began as Clawd, changed to Moltbot and then to OpenClaw in January 2026, before going viral weeks later.

Users can build and deploy AI agents via WhatsApp, Discord, iMessage, Slack, and other popular messaging platforms, with support for multiple underlying AI models including Claude, ChatGPT, Gemini, and Grok. A companion ClawHub skills marketplace lets users download automation skills for tasks like managing email inboxes, trading stocks, and other automated workflows. Supporting multiple platforms and models simultaneously has been a central part of OpenClaw’s appeal to developers building agents on existing communication infrastructure.

Expert Reactions and Security Concerns

That rapid adoption attracted scrutiny as quickly as it gathered stars. Security researcher John Hammond at Huntress described OpenClaw as a wrapper over underlying AI models, arguing that beneath the hype the framework adds little that the models themselves do not already provide:

“At the end of the day, OpenClaw is still just a wrapper to ChatGPT, or Claude, or whatever AI model you stick to it.”

John Hammond, Senior Principal Security Researcher at Huntress (via TechCrunch)

The assessment reflects a recurring skeptic position among researchers: that OpenClaw is primarily an interface layer over existing AI models rather than a novel technical contribution to the field.

AI researcher Andrej Karpathy offered a different view, pointing to what OpenClaw enables rather than its underlying architecture. Referring to Moltbook, an AI-agent social network built on OpenClaw’s infrastructure that hosts AI agents interacting on users’ behalf, Karpathy called what he witnessed “the most incredible sci-fi takeoff-adjacent thing” he had seen recently. He cited it as evidence that the framework is already enabling genuinely new forms of networked software.

Karpathy’s enthusiasm has not muted safety concerns, however. Permiso Security CTO Ian Ahl demonstrated that prompt injection vulnerabilities in OpenClaw could allow a compromised agent with access to a user’s email and messaging to take unauthorized actions. Hammond’s advice for everyday users was direct: “Speaking frankly, I would realistically tell any normal layman, don’t use it right now.”

The gap between Hammond’s and Karpathy’s assessments reveals that observers are measuring OpenClaw against fundamentally different standards. Hammond evaluates it as infrastructure, a layer that adds little beyond what the underlying models already provide, and one with exploitable attack surfaces. Karpathy evaluates it as an enabler, judging it by what networked deployments like Moltbook already demonstrate is possible. The security risks are real regardless of which frame applies.

Already the 21st-ranked code repository on GitHub by mid-February, OpenClaw added another wave of stars to overtake React entirely. That shift signals a broader reorientation of developer attention away from traditional UI frameworks and toward AI agent tooling.

Whether the project holds up under production workloads, and whether Steinberger addresses the security vulnerabilities, remains the next test. Hammond’s warning leaves that question open.
